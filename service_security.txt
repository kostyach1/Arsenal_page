# Идеи защиты сервиса:

1. Использовать технологию мультиоблака для хранения данных, построения архитектуры. Важные данные можно хранить в частном облаке (пароли, логины и тд).
2. Продумать вопрос разделения сервиса (чтобы при отказе одной части не ломался весь сервис сразу, а выходила из строя часть или несколько частей).
3. Для защиты данных потребуется регулярный мониторинг системы и отслеживание подозрительной активности.
4. Периодическая проверка запущенных программ, чтобы сразу обнаруживать незнакомые процессы и прекращать исполнение сторонних приложений.
5. Использование одного из провайдеров для хостинга сервисов, их услуг по защите. 

# Вероятные атаки: 

-- Межсайтовый скриптинг (XSS): - наиболее популярная
1. Правильное кодирование, включая кодирование HTML, атрибутов, кодирование JavaScript, CSS и т.д.
2. Установка брандмауэров, действующих как фильтр, который выявляет и блокирует любые вредоносные запросы к веб-сайту
3. Отслеживание за вводимыми пользователем данными, отсутствие скриптов.
4. Санитизация данных

-- Атака с использованием SQL-инъекции:
1. Использование параметризованных запросов, аутентификации для защиты базы данных
2. Проверка входных данных по белому списку портов и хостов
3. Использование подходящей библиотеки ORM (Hibernate, Entity Framework, ActiveRecord и др., в зависимости от платформы)
4. Защита с использованием PDO или prepared statements.
5. Ограничение прав доступа к базе данных сайта.

-- Отказ в обслуживании (DoS-, DDoS-атаки):
1. Регулярная проверка трафика (на увеличение)
2. Если есть значительное увеличение трафика,  можно большую часть трафика перенаправлять с сервера
3. Установка услуги защиты сайта от взломов (WAF)
4. Использование шаблонов для фильтрации ботового и реального трафика 

# Рассмотрим социальную сеть Одноклассники

-- Боты:
Боты могут создавать ненужную активность
1. Требовать номер телефона при регистрации, не разрешать регистрировать несколько аккаунтов на один номер, ограничивать количество и частоту попыток ввода кода подтверждения с одного IP-адреса
2. Двухфакторная аутентификация, возможно
3. Ввести ограничение на число постов за определенный промежуток времени

-- Фишинг:
Мошенничество, размещение ссылок с опасным содержимым.
1. Блокировка ссылок на непроверенные ресурсы
2. Блокировка ссылок на ресурсы с истекшим сертификатом безопасности
3. Двухфакторная аутентификация для защиты аккаунтов
4. Блокировка пользователя, разместившего опасную ссылку

-- Борьба с фейками:
1. Обеспечить верифицированным пользователям более надежную защиту (усложнить процедуру входа в аккаунт)
2. Верифицировать аккаунты только после предоставления доказательств совпадения личности (вероятно, скан документов, фото с разных ракурсов) 
3. Блокировать использование галок в именах пользователей 
4. При попытках изменения данных верифицированного аккаунта, выполнить проверку личности
5. Блокировка пользователей с одинаковым ником и похожей аватаркой с верифицированным пользователем
